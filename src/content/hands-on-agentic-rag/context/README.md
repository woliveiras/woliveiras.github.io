# Agentic RAG Series Context Index

Use this directory as the operational context for agents working on the Hands-on Agentic RAG series.

Do not load `../brainstorming.md` by default. It is historical source material. Start with `../instructions.md`, then use this index to load only the context needed for the task.

## Recommended Reading Order

For any task:

1. `../instructions.md`
2. `context/README.md`
3. The task-specific files listed below

## Task Routing

Writing a blog post:

- `series-positioning.md`
- `editorial-guidelines.md`
- `domain-and-safety.md`
- `series-outline.md`
- The relevant section in `post-briefs.md`

Reviewing a blog post:

- `editorial-guidelines.md`
- `domain-and-safety.md`
- `series-outline.md`

Planning or implementing VetSupport:

- `technical-architecture.md`
- `domain-and-safety.md`
- `project-briefs.md`

Planning the series roadmap:

- `series-positioning.md`
- `series-outline.md`
- `writing-roadmap.md`

Checking whether a new idea fits the series:

- `series-positioning.md`
- `domain-and-safety.md`
- `technical-architecture.md`

## File Map

- `series-positioning.md`: title, promise, audience, differentiation, and VetSupport narrative.
- `editorial-guidelines.md`: voice, post structure, MDX rules, and quality checklist.
- `domain-and-safety.md`: veterinary domain boundaries and safety rules.
- `technical-architecture.md`: local agent harness, stack, retrieval, LLM, and observability decisions.
- `series-outline.md`: compact outline for all parts and posts.
- `post-briefs.md`: objective and key ideas for each planned post.
- `project-briefs.md`: practical VetSupport projects and expected outputs.
- `writing-roadmap.md`: writing phases and deliverables.

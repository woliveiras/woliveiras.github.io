# Series Positioning

## Title

**Hands-on Agentic RAG: building reliable agents for complex scenarios**

Subtitle idea:

**Developing AI agents with RAG for sensitive domains, with a focus on safety, traceability, and decision support.**

## Promise

Teach readers how to build agents that retrieve knowledge, organize history, cite evidence, respect privacy, and support work without replacing professionals.

## Practical Thread

The series builds **VetSupport**, a local agent harness for veterinary clinics.

VetSupport is not a SaaS product in the main path. It is a practical learning system that lets readers seed data, ingest documents, run retrieval, execute agents, evaluate outputs, and inspect what changed.

## Core Thesis

RAG in AI agents is not just semantic search. In sensitive domains, RAG is infrastructure for knowledge access, memory, safety, traceability, and decision support.

## Audience

Write for:

- AI Engineers.
- Software Engineers working with LLMs.
- Product Engineers building AI features.
- Developers experimenting with LangGraph, LlamaIndex, LangChain, or similar frameworks.
- Technical founders building pet tech, B2B SaaS for clinics, or products in regulated domains.
- Applied researchers interested in RAG, agents, and evaluation.

Assume the reader understands Python, APIs, databases, basic LLM concepts, embeddings, and software architecture. Do not write for absolute beginners.

## Differentiation

The series is not "learn RAG with Python." The difference is:

> Learn to build reliable RAG agents for complex scenarios where context, safety, privacy, and traceability matter.

The series should feel above simple demos because it deals with real engineering problems:

- Demos of RAG are easy.
- RAG in production is hard.
- Agents make mistakes.
- Documents are messy.
- Sensitive data requires care.
- Evaluation is still poorly understood.
- Veterinary clinics deal with fragmented history, physical documents, exams, and communication with tutors.
- AI products need to be reliable.

## VetSupport Narrative

Across the series, the reader evolves VetSupport:

1. Seed clinic data.
2. Add documents.
3. Parse documents.
4. Create domain-aware chunks.
5. Index chunks.
6. Add metadata.
7. Add structured retrieval.
8. Add timelines.
9. Add the agent orchestrator.
10. Add safety checks.
11. Add evaluation.
12. Add observability.
13. Add privacy and permissions.
14. Run the system end to end through the local agent harness.
